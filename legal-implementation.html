SQL — Supabase Dashboard → SQL Editor → New query
Record when each installer accepted your Terms
Adds two columns to your installers table: when they accepted, and which version they accepted. If you ever update the Terms, you can see which installers have seen the new version and which haven't. Without this, you have no proof of consent if it's ever disputed.
1
Add terms_accepted_at and terms_version columns to installers
These two columns record the exact moment an installer ticked the checkbox at signup, and which version of the Terms they agreed to. Existing installers get a NULL — you will prompt them to accept the next time they log in (see Code B).
SQL Non-destructive — adds columns only
Supabase SQL Editor 
-- Add terms acceptance tracking to the installers table
ALTER TABLE installers

  -- Timestamp of when they ticked the checkbox
  ADD COLUMN IF NOT EXISTS terms_accepted_at
    TIMESTAMPTZ DEFAULT NULL,

  -- Which version of the Terms they accepted ("v1.0 May 2026")
  ADD COLUMN IF NOT EXISTS terms_version
    TEXT DEFAULT NULL,

  -- Their IP address at the time of acceptance (optional but useful)
  ADD COLUMN IF NOT EXISTS terms_accepted_ip
    TEXT DEFAULT NULL;

-- Confirm the columns were added
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'installers'
  AND column_name IN ('terms_accepted_at', 'terms_version', 'terms_accepted_ip');
-- Expected: 3 rows returned
  • Existing 16 installers will have NULL for these columns — you will prompt them to accept when they next log in (Code B below handles this)
  • Every new installer who signs up will have a precise timestamp and version recorded the moment they tick the checkbox
  • When you update the Terms in future, change the version string (e.g. "v1.1 August 2026") and the system will know who has seen the new version

ALSO ADD — this JavaScript function anywhere in your <script> block
// Keeps the signup button disabled until both checkboxes are ticked
function checkConsentReady() {
  const termsOk = document.getElementById('consent-terms').checked;
  const dataOk  = document.getElementById('consent-data').checked;
  const signupBtn = document.getElementById('signup-btn'); // ← use your button's actual ID
  if (signupBtn) {
    signupBtn.disabled = !(termsOk && dataOk);
    signupBtn.style.opacity = (termsOk && dataOk) ? '1' : '0.45';
    signupBtn.style.cursor  = (termsOk && dataOk) ? 'pointer' : 'not-allowed';
  }
}

// Call once on page load so button starts disabled
document.addEventListener('DOMContentLoaded', () => checkConsentReady());
  • The submit/register button is greyed out and unclickable until both boxes are ticked
  • Both documents open in a new tab so the installer can actually read them before accepting
  • The checkbox IDs (consent-terms, consent-data) are used in Code B below to record the acceptance

// ── Add this function anywhere in your script block ──────────
function showTermsAcceptanceModal(installerId) {
  // Create a full-screen overlay
  const overlay = document.createElement('div');
  overlay.id = 'terms-modal';
  overlay.style.cssText = `
    position:fixed;inset:0;z-index:9999;
    background:rgba(7,9,14,.97);
    display:flex;align-items:center;justify-content:center;padding:24px`;

  overlay.innerHTML = `
    <div style="max-width:520px;background:#111620;border:1px solid rgba(255,255,255,.08);
                border-radius:16px;padding:36px 32px;text-align:center">
      <div style="font-family:'DM Serif Display',serif;font-size:26px;
                  color:#edf1f6;margin-bottom:14px">
        We've updated our Terms
      </div>
      <p style="font-size:14px;font-weight:300;color:#7a90a8;line-height:1.7;margin-bottom:24px">
        Nova Metrics now has formal Terms of Service and a Privacy Policy.
        Please read and accept them to continue using the platform.
      </p>
      <div style="display:flex;gap:12px;justify-content:center;margin-bottom:24px">
        <a href="terms.html" target="_blank"
           style="font-size:13px;color:#c8911c;text-decoration:underline">
          Read Terms of Service ↗
        </a>
        <span style="color:#4a6080">·</span>
        <a href="privacy.html" target="_blank"
           style="font-size:13px;color:#c8911c;text-decoration:underline">
          Read Privacy Policy ↗
        </a>
      </div>
      <label style="display:flex;align-items:flex-start;gap:10px;
                     text-align:left;margin-bottom:20px;cursor:pointer">
        <input type="checkbox" id="modal-consent"
               style="margin-top:3px;accent-color:#c8911c;flex-shrink:0">
        <span style="font-size:13px;font-weight:300;color:#7a90a8;line-height:1.6">
          I have read and agree to the Terms of Service and Privacy Policy.
        </span>
      </label>
      <button id="modal-accept-btn"
              onclick="acceptTermsFromModal(${installerId})"
              disabled
              style="width:100%;padding:13px;border-radius:9px;border:none;
                     background:#c8911c;color:#07090e;font-size:14px;
                     font-weight:500;cursor:not-allowed;opacity:0.45;
                     font-family:'DM Sans',sans-serif;transition:all .2s">
        I accept — continue to platform
      </button>
    </div>`;

  // Enable button only when checkbox is ticked
  overlay.querySelector('#modal-consent').addEventListener('change', function() {
    const btn = overlay.querySelector('#modal-accept-btn');
    btn.disabled = !this.checked;
    btn.style.opacity = this.checked ? '1' : '0.45';
    btn.style.cursor  = this.checked ? 'pointer' : 'not-allowed';
  });

  document.body.appendChild(overlay);
}

async function acceptTermsFromModal(installerId) {
  await supabase.from('installers').update({
    terms_accepted_at: new Date().toISOString(),
    terms_version:     'v1.0 May 2026'
  }).eq('installer_id', installerId);

  // Remove the modal and let them in
  document.getElementById('terms-modal')?.remove();
}
  • Existing installers see a clean modal on their next login — can't dismiss it, can read both documents in new tabs, must tick and click Accept
  • After they accept, the modal never shows again — terms_accepted_at is now set
  • New installers accept at signup (Code A) so they never see this modal at all
